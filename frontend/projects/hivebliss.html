Hivebliss

An e-commerce platform for a honey & herbal plant shop that handles order management, bank transfer payments, and admin dashboards.

Project Overview

Hivebliss is a production-ready e-commerce platform built for a small Sri Lankan business selling raw honey and herbal plants. It is a complete MERN stack application covering everything from product browsing and cart management to a full order lifecycle, automated email notifications, PDF receipt generation, and a comprehensive admin dashboard — all deployed live with a custom domain.

Key Features

Full Shopping Flow

Product listings with search, category filters, and sorting. Full cart management, checkout with address pre-fill, and instant order confirmation with receipt download.

JWT Authentication

Secure register, login, forgot/reset password, and profile management. Separate customer and admin roles with protected routes and middleware guards across all API endpoints.

WhatsApp Payment Flow

Bank transfer payment model with a pre-filled WhatsApp confirmation message — designed for Sri Lanka's local market. PDFKit receipts generated in-memory and stored on Cloudinary.

6-Stage Order Lifecycle

Complete order pipeline: Pending Payment → Payment Received → Verified → Shipped → Delivered, with stock reservation, cancellation handling, and automated email notifications at every stage.

Admin Dashboard

Full admin panel with revenue stats, order management with status actions, product CRUD with Cloudinary image uploads, user management, and low-stock alerts in a sidebar layout.

Public Order Tracking

Anyone can track an order by number without logging in — featuring a visual progress bar, full status timeline, and cancelled order handling. No account required.

Technologies Used

React + Vite

Frontend framework with React Router v6, Context API for auth and cart state, Tailwind CSS for styling, and Lucide React for a consistent icon system throughout the app.

Node.js + Express

RESTful backend API with a modular route structure covering auth, products, cart, orders, and admin operations. Deployed on Render as a persistent web service.

MongoDB Atlas

Cloud-hosted NoSQL database with Mongoose ODM. Schemas for users, products, orders, and carts with virtual fields, middleware hooks, and stock reservation logic.

Cloudinary

Cloud storage for product images and PDF receipts. Multer handles multipart uploads in Express, with fl_attachment transformation URLs for forced browser downloads.

PDFKit

In-memory PDF receipt generation streamed directly to a buffer and uploaded to Cloudinary — no disk writes required, making it fully compatible with serverless hosting.

Nodemailer

Automated transactional email notifications sent to customers and the shop owner at key order lifecycle events using Gmail's SMTP service with app password authentication.

JWT + bcrypt

Stateless authentication with 7-day token expiry, password hashing via bcrypt, and role-based middleware guards protecting all admin-only API routes.

Vercel + Render

Frontend deployed on Vercel with SPA rewrite rules and a custom domain (hivebliss.store). Backend on Render with all secrets managed via environment variables.

Challenges & Solutions

Stock Reservation Under Concurrent Orders

Challenge: Preventing overselling when multiple customers order the same product simultaneously before payment is verified by an admin.

Solution: Implemented a reservedStock field on the Product model. Stock is reserved atomically on order creation and only deducted from actual stock on admin verification — keeping available inventory accurate throughout the entire payment window without blocking legitimate orders.

PDF Receipt Generation & Browser Download

Challenge: Generating formatted PDF receipts in a serverless-friendly environment and delivering them as real file downloads in the browser rather than opening inline in a new tab.

Solution: Used PDFKit with a streaming buffer approach — the receipt is assembled in memory, piped to a buffer, and uploaded to Cloudinary. The frontend fetches the URL, converts the response to a blob, and triggers a native browser download via a temporary anchor element with the fl_attachment Cloudinary transformation.

CORS in a Monorepo Cross-Service Deployment

Challenge: Managing CORS correctly when the frontend and backend share the same GitHub monorepo but are deployed to completely separate services — Vercel and Render — with different origins and dynamic preview URLs.

Solution: Configured a dynamic CORS origin whitelist in Express driven by a CLIENT_URL environment variable, alongside hardcoded localhost entries for development. This allowed both development and production origins to be managed without any code changes between environments.

Adapting to Local Market Payment Constraints

Challenge: Sri Lanka's e-commerce landscape lacks widely accessible card payment gateways — most small businesses rely on bank transfers confirmed via WhatsApp, which no standard payment library supports out of the box.

Solution: Designed a custom order flow where the system generates a pre-filled WhatsApp message containing the order number, total, and bank account details, triggered immediately after checkout. Admins then manually mark payment as received before verifying, fulfilling, and shipping the order through the admin dashboard.

SPA 404 Errors on Page Refresh in Production

Challenge: React Router handles all client-side routing, but Vercel's server looks for actual files at those paths on refresh — causing 404 errors whenever a user refreshed any page other than the root URL.

Solution: Added a vercel.json configuration file with a catch-all rewrite rule that routes all requests back to index.html, allowing React Router to take over and render the correct component for any URL without any server-side file resolution conflicts.

Key Learnings

  • Gained complete end-to-end MERN stack experience building a production application with real users, live data, and a deployed custom domain.
  • Learned to design business logic around local market constraints — adapting payment and fulfilment flows to real-world conditions rather than generic patterns.
  • Developed expertise in Cloudinary integration for both image uploads and binary PDF storage, including delivery transformations for forced browser file downloads.
  • Built a complete JWT authentication system from scratch including role-based route guards, password reset token flows, and secure cookie-free token expiry handling.
  • Understood production deployment challenges — CORS configuration across services, environment variable parity, DNS propagation, and SPA routing on Vercel.
  • Improved skills in designing admin interfaces with real operational value — revenue dashboards, paginated order tables, product CRUD with image management, and order action workflows.
  • Practiced structuring and deploying a monorepo project with separate frontend and backend deployed to different cloud providers from the same GitHub repository.
