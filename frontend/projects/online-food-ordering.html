Online Food Ordering System

A Java-based Food Ordering Management System with Advanced Data Structures

Project Overview

The Online Food Ordering System is a simplified console-based application designed to simulate restaurant order management. Built entirely in Java, this system demonstrates the practical implementation of various data structures including LinkedList, ArrayDeque, PriorityQueue, and Stack. The project showcases efficient order processing with priority handling, dynamic menu management, and comprehensive reporting capabilities.

Key Features

Dynamic Menu Management

Add, remove, and edit menu items dynamically using LinkedList for flexible data management

Priority Order Processing

Handle normal orders via FIFO queue and premium orders with PriorityQueue for efficient service

Undo Functionality

Revert the most recent action using Stack-based implementation for error recovery

Custom Merge Sort

Implemented custom sorting algorithms for efficient data organization and reporting

End-of-Day Reports

Generate comprehensive reports with total bills and popular items sorted by frequency

Performance Logging

Runtime execution tracking using System.nanoTime() for performance analysis

Challenges & Solutions

Priority Queue Management

Challenge: Implementing an efficient system that prioritizes premium customer orders while ensuring normal orders are not indefinitely delayed. The system needed to balance fair service with priority handling.

Solution: Utilized Java's PriorityQueue with a custom Comparator to assign priority levels to orders. Premium orders were given higher priority values, ensuring they were served first while normal orders were processed through a separate FIFO ArrayDeque to maintain order fairness.

Undo Action Implementation

Challenge: Creating a reliable undo mechanism that could revert various types of actions (add, remove, edit menu items, and serve orders) while maintaining data consistency.

Solution: Implemented a Stack-based (Deque) command pattern where each action was stored as an object containing the action type and relevant data. The undo operation would pop the last action and execute its reverse operation, ensuring proper state restoration.

Custom Sorting Algorithm

Challenge: Efficiently sorting completed orders by total bill amount and menu items by popularity without using built-in Java sorting methods, as per project constraints.

Solution: Implemented custom Merge Sort algorithm with O(n log n) time complexity. Created separate comparator logic for different sorting criteria (bill amount and sales frequency), with performance logging using System.nanoTime() to verify efficiency.

Data Structure Selection

Challenge: Choosing the most appropriate data structure for each functionality while adhering to project constraints that limited available data structures.

Solution: Carefully analyzed the operations required for each feature. Used LinkedList for menu items (frequent additions/deletions), ArrayDeque for normal orders (FIFO), PriorityQueue for premium orders (priority-based), and Deque as Stack for undo operations (LIFO), optimizing for each specific use case.

Limitations & Future Improvements

Current Limitations

• Supports only single restaurant operations (no multi-branch functionality)
• No data persistence - all data is lost on application restart
• Console-based interface only, lacking graphical user interface
• Limited to sequential processing without multi-threading support
